# 問題フォーマットガイド(解いて学ぶ)

このアプリの **Markdownフォーマット自体** を問題にしたデッキです。
解きながら問題の書き方を覚えられます。詳細はリポジトリの README にもあります。

## 新しい問題を始めるときに使うMarkdownの記法はどれ?

- [x] `## 問題文` (見出しレベル2)
- [ ] `# 問題文` (見出しレベル1)
- [ ] `- 問題文` (リスト)
- [ ] `**問題文**` (太字)

> `##` の見出しが1問の区切りになり、見出しのテキストがそのまま問題文になります。
> `#` (レベル1)はデッキのタイトル用です。

## 選択問題の正解の選択肢の書き方として正しいものは?

- [x] `- [x] 選択肢のテキスト`
- [ ] `- [o] 選択肢のテキスト`
- [ ] `- (正解) 選択肢のテキスト`
- [ ] `* 選択肢のテキスト ←正解`

> チェックリスト記法で `[x]` を付けた選択肢が正解になります。
> `[x]` を2つ以上付けると、自動的に「複数選択問題」になります。

## `[x]` の選択肢を2つ以上書くと複数選択問題になる

答え: ○

> 正解が1つなら四択(択一)、複数なら「すべて選べ」形式として自動判定されます。

## 並び替え問題を作る手順を正しい順に並べ替えよ

1. `##` で問題文を書く
2. 正しい順番で番号付きリスト(`1.` `2.` …)を書く
3. 必要なら `>` で解説を書く

> 番号付きリストを**正解の順番**で書くだけで並び替え問題になります。
> 出題時には自動でシャッフルされます。

## 正誤(○×)問題は「答え: ○」または「答え: ×」の行で作る

答え: ○

> `答え: ○` / `答え: ×` のほか、`o` `x` `正` `誤` `true` `false` なども使えます。

## 解説を書くときに使う記法はどれ?

- [ ] `<!-- 解説 -->` (コメント)
- [ ] `[解説]: ...`
- [x] `> 解説のテキスト` (引用)
- [ ] `~~解説のテキスト~~` (打ち消し線)

> 引用記法 `>` で書いた部分が解説として、回答後に表示されます。
> 複数行書いてもOKで、**太字**や `コード` などのMarkdown記法も使えます。
