# 🚃 Learning Express

Markdown ファイルを元データにした、GitHub Pages で動く学習クイズアプリです。
ビルド不要・依存ライブラリなしの静的サイトなので、**md ファイルを追加して push するだけ**で問題集が増えます。

## できること

- **四択(択一)問題** — 選択肢をタップで即judge
- **複数選択問題** — `[x]` を複数付けるだけで「すべて選べ」形式に
- **並び替え問題** — ▲▼ボタンで順番を組み立てて回答
- **正誤(○×)問題** — ○/× の2択
- **フラッシュカード(自己採点)** — 単語リストや穴埋めを「めくって自分で○×判定」
- **2種類の書き方に対応** — かんたん形式と、一問一答・単語リスト・穴埋めの**教材形式**
- 回答ごとの**正誤判定と解説表示**
- 結果画面(スコア・所要時間・全問の振り返り)
- **章(カテゴリ)ごとに出題範囲をしぼる**フィルタ
- 問題数の多いデッキは**ランダム20問**の腕試しモード
- **間違えた問題だけを復習**するモード(履歴はブラウザの localStorage に保存)
- 出題シャッフル/選択肢の自動シャッフル
- デッキごとの成績記録(挑戦回数・前回・自己ベスト)
- ダークモード、キーボードショートカット(`1`〜`4` で選択、`Enter` で次へ、○×は `o` / `x`、カードは `Enter` でめくる)
- スマホ対応レイアウト

## 公開手順(初回のみ)

1. このブランチを `main` にマージする
2. リポジトリの **Settings → Pages → Build and deployment → Source** を **GitHub Actions** にする
3. `main` に push されるたびに自動デプロイされ、`https://<ユーザー名>.github.io/Learningexpress/` で公開されます

## 問題の追加方法

1. `data/` に Markdown ファイルを作る(下のフォーマット参照)
2. `data/index.json` の `decks` 配列にファイルパスを追記する

```json
{
  "decks": [
    "data/sample-it.md",
    "data/my-new-deck.md"
  ]
}
```

3. commit & push → 自動デプロイ

> ローカルで確認する場合は `python3 -m http.server` などで配信して `http://localhost:8000` を開いてください(`file://` 直開きでは fetch が動きません)。

## Markdown フォーマット

1ファイル = 1デッキ。`#` がデッキ名、`##` が問題の区切りです。
問題形式は書き方から自動判定されます。サンプルは [`data/sample-it.md`](data/sample-it.md) を参照してください。

````markdown
# デッキのタイトル

ここに書いた文章はデッキの説明になります。

## 四択問題の問題文をここに書く

- [x] 正解の選択肢
- [ ] 不正解の選択肢
- [ ] 不正解の選択肢
- [ ] 不正解の選択肢

> ここに解説を書く(回答後に表示。省略可)

## 複数選択は [x] を2つ以上付けるだけ

- [x] 正解その1
- [x] 正解その2
- [ ] 不正解

## 並び替え問題は番号付きリストを「正解の順番」で書く

1. 最初にやること
2. 次にやること
3. 最後にやること

> 出題時には自動でシャッフルされます。

## 正誤問題は「答え:」の行を書く(この文が正しいか問う)

答え: ○

> ○/×のほか o, x, 正, 誤, true, false でもOK。

## 問題文の補足・コード・画像も書ける

見出しの下に書いた段落は問題文の続きとして表示されます。

```javascript
console.log("コードブロックもOK");
```

- [x] 選択肢には **太字** や `コード` も使える
- [ ] 画像は `![alt](url)` で挿入できる

> 解説も複数行・Markdown記法OKです。
````

### 判定ルール早見表

| 書き方 | 形式 |
| --- | --- |
| `- [x]` / `- [ ]` のチェックリスト | 選択問題(`[x]`が1つ=択一、2つ以上=複数選択) |
| `1.` `2.` … の番号付きリスト | 並び替え(書いた順=正解) |
| `答え: ○` / `答え: ×` の行 | 正誤(○×)問題 |
| `>` の引用 | 解説(回答後に表示) |

形式を判別できない問題はスキップされ、ブラウザの開発者コンソールに警告が出ます。

## 教材形式(一問一答・単語リスト・穴埋め)

「かんたん形式」とは別に、参考書のような**一問一答・単語リスト・穴埋め**の書き方もそのまま読み込めます。
実データの [`不動産登記法_本試験対策.md`](不動産登記法_本試験対策.md) がこの形式です(105問を自動抽出)。

````markdown
# デッキタイトル

## 第1章 総論          ← ## や ### は「章・節」になり、出題範囲フィルタに使われます

### 1-1 一問一答(○×)

**Q1.** 表示に関する登記は、申請がなくても登記官が職権ですることができる。
**A.** ○ (不登法28条)

**Q2.** 権利に関する登記は、登記官が職権で行うことが一切できない。
**A.** × 例外として職権でできる場合がある。   ← ×/○ の後ろは解説になります

### 1-2 単語リスト        ← テーブルは「用語→定義」のフラッシュカードになります

| 用語 | 定義 |
|---|---|
| 登記識別情報 | 本人確認の手段として法務局から通知される12桁の符号(2条14号) |

### 1-3 穴埋め            ← 答えが ○/× 以外なら、めくって自己採点するカードになります

**Q.** 登記の申請は、…政令で定める情報(以下「( ① )」という)を…提供してしなければならない。
**A.** ① 申請情報 (18条)
````

| 書き方 | 形式 |
| --- | --- |
| `**Q.**` 〜 `**A.** ○` / `**A.** ×` | 正誤(○×)問題(`A.` の○×の後ろは解説) |
| `**Q.**` 〜 `**A.** それ以外` | フラッシュカード(穴埋め等。めくって自己採点) |
| `\| 用語 \| 定義 \|` のテーブル | フラッシュカード(1列目=表、2列目以降=裏) |
| `##` / `###` の見出し | 章・節(出題範囲フィルタのカテゴリ) |

> **同名の重複ファイルについて**: `不動産登記法_本試験対策-1〜3.md` は本体と中身が同一だったため、`data/index.json` には1本だけ登録しています。章ごとに別教材にしたい場合は、各ファイルを編集して `index.json` に追加してください。

## ファイル構成

```
index.html              アプリ本体(SPA)
css/style.css           スタイル
js/parser.js            Markdown → 問題データ変換
js/storage.js           成績・間違い履歴の保存(localStorage)
js/app.js               出題・判定・画面遷移
data/index.json         デッキ一覧(ここに md を登録)
data/*.md               問題データ
.github/workflows/      GitHub Pages 自動デプロイ
```
